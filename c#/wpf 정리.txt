
1.시작
*1.wpf프로젝트 구조
  디펜던시는 의존성폴더
  
  app.xaml은 어플리케이션의 시작점
  여기서 시작할 찾을 startupril로 띄움(기본값 mainWindow.xaml)
  여기서 리소스나 파일을 추가할수도있음
  
  mainWindow.xaml은 기본적인 메인 폼임
  또한 확장하면 mainWindow.xaml.cs가 있는데 여기다가 코드를 적을수있음
  
*2.xaml 디자인  
  텍스트박스의 텍스트는 Test
  버튼의 라벨은 Content
  
*3.xaml 구조
   실제로 우리가 작성하게되는 xmal은 Grid사이에 작성하게됨
   맨위에 window는 해당 폼 자체에 대한 설정이고,여기서 가로세로를 지정할수있고,이름을 정할수있음
   
   Grid는 그리드 내부요소의 레이아웃을 제어하기 위한것
  
2.WPF 윈도우
*1.속성 창 
  라이더에선 지원안함
*2.window 속성 종류
  속성은 컨트롤 스페이스로 해당 속성리스트를 띄울수있음 Cursor=""에서 ""에 포커스 주고 컨트롤스페이스하면 됨
  
  Title은 창의 이름
  Cursor은 런타임에서 해당 컴포넌트 위에 마우스를 올려놨을때 커서를 바꿔줌(손모양이나 화살표같은거) Cursor="No" 이런거하면 금지모양나옴
  isEnabled로 해당컴포넌트를 비활성화할수있음
  WindowState로 시작할때 창의 크기설정(최대화최소화같은거)를 할수있음
  WindowStartupLocation로 창의 위치설정을 할수있음,보통 CenterScreen를 사용해서 정중앙에 띄움
  ShowInTaskbar로 상태표시줄에서 프로그램이 안보이게 할수있는데 어따쓰는진모르겠음
  
  Height와 Width로 창크기조절을 할수있음
  ResizeMode로 크기조절을 막을수있음(최대화 최소화 창크기변경같은거 못하게)
  TopMost로 항상위를 켤수있음
  
*3.백그라운드 변경 
  Background로 배경색을 바꿀수있음,16진수로 rgb+alpha(투명도)로도 할수있음
  그리고 Background하위로 GradientBrush로 여러색을 섞어서 그라데이션도 넣을수있음(LinearGradientBrush,GradentStep)
  이미지를 추가할땐 해당 프로젝트에 이미지를 넣고
      <Window.Background>
        <ImageBrush ImageSource="/data/abc.jpg"> </ImageBrush>
      </Window.Background>
  이런식으로 할수있음
  그리고 Stretch로 이미지크기를 해당컴포넌트 크기에 맞게 늘리거나 채우거나 할수있음(기본값 채우기)
  단 이때 실행파일은 이미지를 찾을수없으니 해당 실행파일이 있는곳에 경로에 맞게 이미지를 넣어야함
  
*4.보더 변경  
  BorderBrush와 BorderThickness를 같이 설정하면 해당 컴포넌트의 테두리색과 테두리크기를 지정할수있음(하나만하면안됨)
  또한 상하좌우를 각각 설정할수도있고,통일할수도있음(7,0,0,0 이나 7 이런식으로 하면됨)
  
*5.폰트 사용  
  상위의 컴포넌트에 폰트나 크기를 지정하면 하위도 자동으로 따라가게됨
  FontStyle="Italic" 등으로 기울임등을 넣을수있고
  FontSize="30"로 크기조절
  FontWeight="Bold"로 볼드등을 넣을수있음
  FontFamily="Century"로 폰트를 지정할수있음
  
*6.창의 크기
  창을 고정픽셀로 설정할수있음
  그리고 창의 최소크기 최대크기도 설정할수있음(MinWidth MaxWidth)

*7.창 아이콘
  창 아이콘에 커스텀아이콘을 설정하려면 Icon에 경로를 넣으면됨
  또한 아이콘을 못찾으면 만들거나 iconfinder.com에서 받을수도있음
  
*8.타이틀 순서변경
  쓸일없을듯
  
*9.컴포넌트 숨기기
  Visibility="Hidden"로 해당 컴포넌트를 숨기고,Visibility="Visible"로 보이게할수있음(기본값)
  Visibility="Collapsed"도 있는데 이건 width와 height도 0으로 설정해버리고,Hidden은 그냥 보이지만 않게하는거
  
  
  
3.일반적인 컴포넌트 1
*1.Grid  
  그리드는 가장 일반적인 레이아웃관리자이자 관리요소임
  각 윈도우에는 최소 하나의 그리드가 있어야하고,그안의 모든것들은 그리드안에 들어있어야함
  
  그리드도 백그라운드색,커서,isEnabled등 기본속성은 다 있음
*2.Button
  버튼도 기본속성들은 다 있고,
    <Button Name="Abc">
  로 이름을 지정할수있음
  
  HorizontalContentAlignment="Center"로 텍스트 가로 위치조절가능
  VerticalContentAlignment="Center"로 텍스트 세로 위치조절가능

*3.Label
  라벨은 그냥 텍스트임
  이것도 이름지정할수있고 할수있는거 다 할수있음
  ToolTip에 값 입력하면 마우스오버했을때 텍스트띄울수있음

*4.TextBox
  텍스트박스는 텍스트를 넣을수있는 박스임
  텍스트박스는 유저가 그 안의 텍스트를 건드릴수있음(즉 복사가 가능,레이블은 불가능)
  기본설정은 똑같고
  
  CaretBrush로 텍스트안에서 I모양 깜빡이는 그거 추가하고 색도 변경할수잇음(현재 텍스트위치 말하는 그거)
  TextWapping로 텍스트가 여러줄가게할지 한줄로 가게할지를 정할수있음(줄바꿈)
  AcceptsReturn으로 엔터를 허용해서 다음줄로 넘기게 할수있음
  VarticalScrollBarVisibility="Auto"로 세로줄이 길때 스크롤바를 추가할수있음
  HorizontalScrollBarVisibility="Auto"로 가로줄이 길때 스크롤바를 추가할수있음
  CharacterCasing으로 대소문자고정을 할수있음
  IsReadOnly로 읽기전용설정가능
  IsUndoEnabled로 실행취소가능
  MaxLength로 텍스트최대길이 설정가능

*5.ComboBox
  콤보박스는 여러 선택지중 하나를 선택하는 박스임
  콤보박스에 아이템을 추가할땐 ComboBoxItem을 가지고 추가할수있음(콤보박스태그안에 추가하면됨)
  또 콤보박스에도 텍스트박스처럼 선택도되고 텍스트를적게할수도있고(IsEditable),선택은 되지만 적진못하게(IsReadOnly)할수도있음
  기본값을 표시할수도있음(SelectedIndex="0")
  
  IsDropDownOpen을 true로 두면 드롭다운메뉴가 창을 열자마자 자동으로 열려있음

4.MDI와 SDI
*1.MDI와 SDI란?
  SDI는 단일창 프로젝트(계산기등)고,MDI는 다중 창 프로젝트(워드,sql서버등)임
  즉 한 창 내에서 끝나면 SDI,여러 창을 뽑을수있으면 MDI임
  기본적으로는 SDI고,MDI로 변경할땐 프로젝트 우클릭해서 add에서 new Window하면됨
  그리고 App.xaml에서 StartUpUri값을 변경하면 기본창을 변경할수있음

                    <TransformGroup>
                        <ScaleTransform></ScaleTransform>

*1.이벤트란?
  이벤트는 특정 동작에 의해 발생하는 콜백함수임(대표적으로 클릭)
  이벤트는 xaml.cs파일에 작성하게됨

*2.이벤트사용법
  윈도우의 메시지박스를 띄우려면
    System.Windows.MessageBox.Show("abc");
  이렇게 쓰면됨
  
  해당 창의 이름을 바꿀땐
    this.Title="abc";
  이렇게하면됨
  
  그룹(그리드)레벨에서 Enabled같은 속성을 바꾸면 안의 모든것들이 영향받음
  이때 주의점은 개별로 변경했던거도 같이 묶여버린다는것

7.디자인 응용 2
1.마진
  마진을 가지고 해당컴포넌트의 위치를 지정할수있음(윈폼에서의 xy픽셀값이라고 볼수있음)
  특이한점은,상하좌우 4개가 있고,0이면 반영하지않는다는것
  이건 정확한 중심점을 찾는게 아닌,해당 컴포넌트와 해당방향의 끝까지의 픽셀수이기때문에,3개이상의 마진을 등록한후에 창크기를 늘리면,
  컴포넌트 크기도 변동됨(마진은 고정되어있기때문에,대신 컴포넌트의 크기가 바뀌는것)
  
  이걸 활용해서 반응형으로 화면을 만들수있음
  또한
	HorizontalAlignment=
	HorizontalAlignment=
  를 사용해서 정렬을 할수있고,해당 영역을 전부 차지하게만들수도있음(Stretch)
  단 이떈 ,Width나 Height의 해당하는걸 삭제해야함

8.이미지와 파일열기
*1.이미지 컴포넌트
  이미지 컴포넌트는 Image를 사용하면됨
  패널같은 컴포넌트는 Rectangle를 사용하면됨,단 이건 그룹핑을 해주는게 아닌,그냥 꾸미기요소임
  
  이미지는 Source로 경로,Stretch로 늘리기선택,StretchDirection으로 늘리기방향을 선택할수있음
  
*2.코드로 이미지로드
  코드로 이미지를 로드할땐
    this.이미지컴포.Source=new BitmapImage("경로");
  이렇게하면됨
  경로는 uri로 하드코딩할수도있고,
    System.AppDomain.CurrentDomain.BaseDirectory+파일이름;
  로 현재 실행경로를 넣을수도있음
  
*3.파일열기
  파일을 열떈
    Microsoft.Win32.OpenFileDialog ofd=new Microsoft.Win32.OpenFileDialog();
	ofd.Filter="Picture|*.jpg;*.png|Text|*.txt" //"이름|확장자1;확장자2|이름2|확장자3"
	ofd.showDialog()
  로 파일선택상자를 열수있음(필터는 선택,없으면 전체파일로 나옴)
  이후에
    ofd.FileName
  등으로 선택한 결과를 받아서 사용할수있음(동기식으로 동작하니까,파일다이얼로그가닫히기전까진 멈춰있음)
  이때 파일이 선택됐는지를 if로 걸러서 사용하는게좋음

	this.textbox1.TextWrapping=TextWapping.Wrap

10.wpf 리소스
*1.윈도우 리소스
  리소스는 어플리케이션 내부및 외부에서 값과 속성을 공유하는방법임
  즉 내부(윈도우리소스)에서보면 해당범위 내에서 전체적으로 속성을 공유하는(폰트등) 방법임(강제가 아님,해당값에 대해 변수로 선언하고 가져다쓴다에 가까움)
  외부(어플리케이션리소스)에서는 서로 다른 윈도우간의 값이나 속성을 공유할떄 유용함
  
  윈도우리소스는 윈도우태그안에
    <Window.Resources>
	  공유속성
	  <SolidColorBrush x:Key="buttonBackground" Color="#FFFFFF00"/>
	  <System:Double x:Key="all_ButtonHeight">45</System:Double>  //이렇게 숫자는 변수처럼 선언하면됨
    </Window.Resources>
  이런식으로 선언하고
    <Button Background="{DynamicResource buttonBackground}">
  이런식으로 사용하면됨

*2.어플리케이션 리소스
  어플리케이션 리소스는 App.xaml에 <Application>태그사이에 적어서 전역변수처럼 쓸수있음
    <Application.Resources>
      ..공유속성
	  <SolidColorBrush x:Key="all_buttonBackground" Color="#FFFFFF00"/>
	</Application.Resources>
  이렇게 똑같이 쓰면됨
  사용도 똑같음
    <Button Background="{DynamicResource all_buttonBackground}"
  이렇게쓰면됨
  
*3.코드로 리소스사용하기
  리소스를 찾을땐
    object ob=this.TryFindResource("리소스명") ; //없으면 null들어있음
  이렇게 받아서 null,타입검사하고 가져다가 쓸수있음
  Double넣을땐 타입캐스팅해서 넣어야함
  
  윈도우리소스를 저장할땐(현재 있는 리소스의 값을 변경할땐)
    this.Resources["리소스명"]=값
    this.Resources["더블리소스명"]=(Double)30
  이렇게 넣으면됨

13.바인딩
*1.바인딩 슬라이더
  바인딩을 사용하면 코드를 사용하지않고 변화점을 서로 바인딩할수있음
  바인딩은
    바인딩소스:값을 읽어올요소
	바인딩유형:바인딩의 유형(일방통행,쌍방통행,역방향일방통행)
	바인딩대상:값이 변경되는 요소
  으로 구성됨
  바인딩객체에서 일방통행은 한쪽의 변화만 뿌리는거고,쌍방통행은 양쪽중 한쪽이 변하면 다른한쪽도 같이변하는거(볼륨)
  
  바인딩은 해당 객체의 해당값에 
    {Binding 해당 속성,ElementName=해당컴포넌트,Mode=바인딩유형,StringFormat=출력포멧}
    <TextBox Text="{Binding Value,ElementName=abcSlider,Mode=OneWay,StringFormat=\{0:F0\}}"/>
  이런식으로 사용하면됨
  또한 양방향을 할땐 모드를 twoWay로 바꿔도되고,양쪽을 단방향으로 둘다 만들어도됨

*2.바인딩 체크박스
  체크박스가 선택되면 특정 그룹박스등의 활성화등을 조절할때도 바인딩을 사용하면 편함
  이땐 일방통행으로 보통 설정하고
    isEnabled="{Binding IsChecked,ElementName=abcCheckBox,Mode=OneWay}"
  이렇게 똑같이하면됨

*3.바인딩 라벨
  라벨에 바인딩할떄도 거의 똑같은데,Text가 아닌 Content에 넣으면됨
  여기서 신경써야할건,StringFormat이 동작하지않으니까 ContentStringFormat에 원하는 출력형식을 지정해줘야함(라벨쪽에서 하드코딩으로 지정)

14.리스트뷰
*1.리스트뷰
  리스트뷰는 리스트박스와 비슷하지만 열이 더 많음
  보통 리스트뷰는 리스트뷰안에 그리드뷰를 넣는식으로 사용됨
  그리드뷰없이 <ListViewItem>을 사용해서 직접 값을 넣어도됨
  
  그리드뷰를 사용하면 컬럼과 로우를 나눠서 사용하기좋음
  그리드뷰를 쓸땐 <GridView>안에 <GridViewColumn>을 넣어서 컬럼을 만들면됨
  각 컬럼별로 값을 할당할땐 리스트안의 값을 나타내는 클래스를 만들고,
  추가할곳에서 해당 클래스로 리스트를 만든다음,리스트를 채우고,리스트뷰의 ItemsSource를 해당 리스트로 등록하면됨
  이러면 기본값으로는 이름매칭해서  같은이름으로 채우는듯
  수동으로 설정할땐,GridViewColumn에
    <GridViewColumn Header="abcd" DisplayMemberBinding="{Binding 해당속성명}" >
  으로 매핑하면되는듯
  이때 출력형식설정은 저번이랑똑같이 StringFormat을 이용하면됨
  
  이미지 출력은 GridViewColumn.CellTemplate의 DataTemplate안에 Image태그에서 Source="{Binding 경로}"를 사용하면됨
  
  
  리스트뷰안에 이미지버튼같이 여러컴포넌트를 한컬럼에 넣고싶을땐,리스트뷰아이템안에 StackPanel을 사용해서 넣으면됨
  코드로 넣을땐 ListViewItem을 생성하고,스택패널을 생성한후 스택패널을 채우고(sp.Children.Add(...))
  리스트뷰아이템에 content를 스택패널로 지정해주면됨
    
*2.리스트뷰 이벤트
  리스트뷰안의 아이템을 선택했을때(또는 선택이 변경됐을때)의 이벤트는
    abcListView_SelectionChanged
